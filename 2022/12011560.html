数据结构

第一章、绪论

0.0

1.1、数据结构的基本概念

  1. 数据：描述客观事物的数、字符以及所有能输入计算机中被计算机程序处理的符号的集合
  2. 数据元素：是数据的基本单位，通常作为一个整体进行考虑和处理(例如：一个班级中的每个学生就是一个数据元素)。一个数据元素可由若干个数据项组成。数据项是构成数据元素的不可分割的最小单位。
  3. 数据对象：性质相同的有限个数据元素的集合，它是数据的一个子集。在默认情况下，数据结构中的数据都是指的数据对象。
  4. 数据结构：所涉及的数据元素的集合以及数据元素之间的关系。因此可以把数据结构看成是带结构的数据元素的集合。数据结构包括如下几个方面：
    1. 数据元素之间的逻辑关系，即数据的逻辑结构，它是数据结构在用户面前呈现的形式
    2. 数据元素及关系在计算机存储器中的存储方式，即数据的存储结构，也叫数据的物理结构
    3. 施加在该数据上的操作，即数据的运算

      1.2、数据结构的三要素

  5. 数据的逻辑结构：
  • 逻辑结构是指数据元素之间的逻辑关系，即从逻辑关系上描述数据,是面向用户的，是独立于计算机的
  • 逻辑结构包括：
    • 集合结构：结构中的数据元素之间除”同属一个集合”外，无其他关系；
    • 线性结构：结构中数据元素之间只存在一对一的关系，除了第一个元素，所有元素都有唯一的前驱。除了最后一个元素，所有元素都有唯一的后继；
    • 树形结构：结构中数据元素之间存在一对多的关系；
    • 图形结构：数据元素之间存在多对多的关系；
  1. 数据的物理(存储)结构：
    存储结构是指数据结构在计算机中的表示(又称映射)，也称物理结构。存储结构包括如下：
    1. 顺序存储：把逻辑上相邻的元素存储在物理位置也相邻的存储单元中，元素之间的关系由存储单元的邻接关系来体现；
    2. 链式存储：逻辑上相邻的元素在物理位置上可以不相邻，借助指示元素存储地址的指针来表示元素之间的逻辑关系；
    3. 索引存储：在存储元素信息的同时，还建立附加的索引表，索引表中的每项称为索引项。
    4. 散列存储：根据元素的关键字直接计算出该元素的存储地址，又称哈希存储；
  2. 数据的运算：施加在数据上的运算包括运算的定义。运算的定义是针对逻辑结构的，指出运算的功能；运算的实现是针对存储结构的，指出运算的具体操作步骤

    1.3、算法的基本概念

  4. 程序=数据结构+算法
  5. 算法是对特定问题求解步骤的一种描述，它是指令的有限序列，其中的每条指令表示一个或多个操作；
  6. 算法的特征：
    1. 有穷性：一个算法必须总在执行有穷步之后结束，且每一步都可在有穷时间内完成。
    2. 确定性：算法中每条指令必须有确定的含义，对于相同的输入只能得到相同的输出。
    3. 可行性：算法中描述的操作都可以通过已经实现的基本运算执行有限次来实现。
    4. 输入：一个算法有零个或多个输入，这些输入取自于某个特定的对象的集合。
    5. 输出：一个算法有一个多个输出，这些输出是与输入有着某种特定关系的量。
  7. 好的算法达到的目标：
    1. 正确性：要求算法能够正确的执行预先规定的功能和性能要求。这是最重要的也是最基础的。
    2. 可使用性:要求算法恩能够很方便的使用
    3. 可读性：算法应该易于人们理解，也就是可读性好。为了达到这个要求，算法的逻辑必须清晰的、简单的和结构化的。
    4. 健壮性：要求算法具有很好的容错性，即提供异常处理，能够对不合理的数据进行检查，不进常出现异常中断或四级现象
  8. 算法的时间复杂度
    1. 指执行算法所需要的计算工作量；
    2. 一个算法花费的时间与算法中语句执行次数成比例。一个算法中的语句执行次数为时间频度，记为：T(n)
    3. 大O表示法：用O( )来体现算法时间复杂度的记法，我们称之为大O表示法。
    4. 一些常见的大O运行时间：
      1. O(log n)，对数时间，二分查找。
      2. O(n)，线性时间，简单查找。
      3. O(n logn)，快速排序——速度较快的排序算法。
      4. O(n²)，选择排序——速度较慢的排序算法。
    5. 个复杂度比较，越小越好：log2n以2为底
      1. O(1)<O(log2n)<O(n)<O(nlog2n)<O(n*n)<O(2^n)<O(n!)
  9. 算法的空间复杂度
    1. 对算法在运行过程中临时占用存储空间大小；
  10. 算法的评估标准：就是算法占用计算机资源的多少

第二章：线性表

2.1、线性表的定义：

  线性表是一种典型的线性结构，也是最常用的一种数据结构。具有相同数据类型的n(n>0)个数据元素的有限序列，其中n为表长，当n=0时，线性表是一个空表;

2.2、线性表的顺序存储结构 — 顺序表

class SqListClass {
    final int initcapacity = 10; //初始表的容量
    public E[] data; //存放元素的数组
    public int size; //表中的元素个数
    private int caption;//存放表的容量,就是当前表的容量

    //初始化表中属性
    public SqListClass() {
        data = (E[]) new Object[initcapacity];
        caption = initcapacity;
        size = 0;
    }

    // 动态的分配空间
    public void UpdataCapacity(int newCapacity) {
        E[] newData = (E[]) new Object[newCapacity];
        // data中元素原封不动的放到新的数组中
        for (int i = 0; i < data.length; i++) {
            newData[i] = data[i];
        }
        caption=newCapacity;//获取当前表的容量
        data=newData;// 仍有data标识数组
    }

    //将一个数组元素插入到data数组中。算法复杂度：O(n)
    public void CreateList(E[] a){
        for (int i = 0; i < a.length; i++) {
            // 出现上溢的情况
            if(size == caption){
                UpdataCapacity( 2*size );
            }
            data[size++]=a[i];
        }
    }

    // 表的基本运算:添加.算法复杂度：O(1)
    public void Add(E a){
        if(size==caption){
            UpdataCapacity( 2*size );
        }
        data[size++]=a;
    }

    // 表的长度
    public int size(){
        return size;
    }

    // 设置表的长度：目的为了缩小空间.算法复杂度：O(1)
    public void SetSize(int nlen){
        if(nlen<0 || nlen>size){
            throw new IllegalArgumentException("设置长度：n不在有限的范围之内");
        }
        size=nlen;
    }

    // 查找：求表中序号为i的元素.算法复杂度：O(1)
    public E GetElem(int i){
        if(i<0 || i>size-1){
            throw new IllegalArgumentException("查找元素：i不在有限的范围之内");
        }
        return data[i];
    }

    //更新：设置下标为i上的值.算法复杂度：O(1)
    public void SetElem(int i,E e){
        if(i<0 || i>size-1){
            throw new IllegalArgumentException("查找元素：i不在有限的范围之内");
        }
        data[i]=e;
    }

    // 查找第一个元素为e的下标.算法复杂度：O(n)
    public int GetElem(E e){
        for (int j = 0; j < size-1; j++) {
            if(data[j].equals( e )){
                return j;
            }
        }
        return -1;
    }

    //data数组通过下标交换两个元素
    public void Swap(int i,int j){
        if((i<0 || i>size-1) || (j<0 || j>size-1)){
            throw new IllegalArgumentException("查找元素：i或j不在有限的范围之内");
        }
        E temp=data[i];
        data[i]=data[j];
        data[j]=temp;
    }

    //在下标为 i 的位置插入元素.算法复杂度：O(n)
    public void InsertElem(int i,E e){
        if(i<0 || i>size){
            throw new IllegalArgumentException("查找元素：i不在有限的范围之内");
        }
        if(size==caption){
            UpdataCapacity( 2*size );
        }
        for (int j = size; j >i; j--) {
            data[j] = data[j-1];
        }
        data[i]=e;
        size++;
    }

    // 删除位置为i 的元素.算法复杂度：O(n)
    public void Delete(int i){
        if(i<0 || i>size-1){
            throw new IllegalArgumentException("查找元素：i不在有限的范围之内");
        }
        for (int j = i; j < size-1; j++) {
            data[i]=data[i+1];
        }
        size--;
        // 满足要求，容量减半
        if(caption>initcapacity && size==caption/4){
            UpdataCapacity( caption/2 );
        }
    }
}

2.3、线性表的特点：

  1. 随机访问(随机存取)，可以在O(1)时间内找到第i个元素
  2. 存储密度高，每个节点只存储数据元素
  3. 扩展容量方便
  4. 插入、删除操作不方便，需要移动大量元素

2.4、线性表的链表存储 — 链表：

  1. 链表的定义：一个节点中不仅包含元素本身的信息，还包含后继或前驱结点的信息。

  2. 如果每个结点只设置了一个指向后继结点的指针成员，称为单链表。当访问过一个结点后，只能接着访问后面的后继结点，无法访问它的前驱结点。

  3. 如果每个结点设置了一个指向后继结点和一个指向前驱节点的指针成员，称为双链表。当访问过一个结点后，可以一次访问后继或前驱结点。

  4. 这里需要注意的是，头结点和首(开始)结点不同，头结点为null，通常不存放任何元素，首结点为第一个元素。

  5. 单向链表不可以 自我删除，在添加和删除的时候都要找到要操作结点的前一个结点

  6. 单链表的添加和删除：

    // 单链表的结点
 class LinkNode {
     E data;// 数据成员
     LinkNode next; //指向的结点地址

     public LinkNode() {
         next = null;
     }

     public LinkNode(E e) {
         data = e;
         next = null;
     }
 }

 // 设置单链表类
 class LinkListClass {
     LinkNode head; // 存放头结点

     public LinkListClass() {
         head = new LinkNode(); // 创建头结点
         head.next = null; // 头结点next为null
     }

     // 使用头插法建表,每次都是从第一个元素插入，所以最后结果是倒叙
     public void CreateListF(E[] a) { // 由数组a整体建立单链表
         LinkNode s; // 插入的结点 的对象
         for (int i = 0; i < a.length; i++) {
             s = new LinkNode<>( a[i] );
             s.next = head.next;
             head.next = s;
         }
     }

     // 使用尾插法建表。这里需要知道，结点是不可以动的，只可以代替，这里t就是为了代替结点。时间复杂度O(n)
     public void CreateListR(E[] a) {
         LinkNode s,t; // 插入的结点 的对象
         t=head;
         for (int i = 0; i < a.length; i++) {
             s=new LinkNode<>(a[i]);
             s.next=t.next; //s的指针指向t.next的结点
             t=s; // t替换为s结点
         }
         t.next=null;
     }

     // 返回序号为i的结点,单链表只可以从头结点开始一个一个向后面找元素
     public LinkNode geti(int i){
         LinkNode p=head;
         int j=-1;
         while (jsize()-1){
             throw new IllegalArgumentException("找出结点：i的范围出错");
         }
         LinkNode p=geti( i-1 );
         p.next=p.next.next;
     }
 }

    2.5、双链表

  7. 双链表中每个结点有两个指针成员，一个指向后继结点，一个指向前驱结点。

双链表的相关操作：

    class DLinkNode { //双链表结点泛型类
        E data; //结点元素值
        DLinkNode prior; //前驱结点指针
        DLinkNode next; //后继结点指针

        public DLinkNode() {
            prior = null;
            next = null;
        }

        public DLinkNode(E data) {
            this.data = data;
            prior = null;
            next = null;
        }
    }

    class DLinkListClass { //双链表泛型类
        DLinkNode dhead; // 存放头结点

        public DLinkListClass() {
            dhead = new DLinkNode<>();
            dhead.prior = null;
            dhead.next = null;
        }

        // 使用尾插法建表
        public void CreateListR(E[] a) {
            DLinkNode s, t;
            t = dhead; // 这里解释一下：链表中的结点是不可以移动的，所以需要使用所谓的'替身'
            for (int i = 0; i < a.length; i++) {
                s = new DLinkNode<>( a[i] );
                t.next = s; //t的后继结点指向s
                s.prior = t; // s的前驱结点指向t
                t = s; //t 替换 s
            }
            t.next = null; //将尾结点的next成员设置为null
        }

        // 双链表的长度
        public int size() {
            DLinkNode t;
            t = dhead;
            int sum = 0;
            while (t.next != null) {
                sum++;
                t = t.next;
            }
            return sum;
        }

        // 双链表的指定元素,这里的i是序号，从0开始
        public DLinkNode geti(int i) {
            DLinkNode p = dhead;
            int j = -1;
            while (j < i) {
                j++;
                p = p.next;
            }
            return p;
        }

        // 在双链表中插入元素
        public void Insert(int i, E e) { // 在序号为i的位置上插入元素
            if (i < 0 || i > size()) {
                throw new IllegalArgumentException( "插入：位置i不在有效范围内" );
            }
            DLinkNode s = new DLinkNode( e );//建立新的结点
            DLinkNode p = geti( i - 1 ); //找到序号为i-1的元素，在其后面加入新的元素s
            s.next = p.next;
            if (p.next != null) { // 判断p结点是否有后继结点
                p.next.prior = s;
            }
            p.next = s; //p的后继结点指向s
            s.prior = p;// s的前驱结点指向p
        }

        // 删除结点
        public void Delete(int i) {
            if (i < 0 || i > size() - 1) {
                throw new IllegalArgumentException( "删除：位置i不在有效范围内" );
            }
            DLinkNode p = geti( i );// 找到指定的元素
            if (p.next != null) {
                p.next.prior = p.prior;
            }
            p.prior.next = p.next;
        }
        // 以上两种算法的时间复杂度都是O(n)
    }

2.6、单链表和双链表的区分

  1. 在单链表中删除一个结点需要找到其前驱结点，而在双链表中删除结点不必找到其前驱结点，在需要找到要删除的结点即可。

    2.7、循环链表

  2. 介绍：循环链表时另一种形式的链式存储结构，分为循环单链表和循环双链表两种形式，分别是从单链表和双链表变化而来。

  3. 循环单链表：

    1. 表中尾结点的next指针域不再是null,而是指向头结点，从而整个链表形成一个首尾相接的表。

    2. 特点：从表中的任一结点出发都可以找到其他节点，与单链表相比，无须增加存储空间。默认情况下，循环单链表也是通过头结点head标识的。

      class LinkListClass {
  LinkNode head; // 存放头结点

  public LinkListClass() {
      head = new LinkNode(); // 创建头结点
      head.next = head; // 置为空的循环单链表
  }
}

    3. 循环单链表的插入和删除与单链表的相同，所以两者的许多基本运算算法是相似的，只要区别就是：

      1. 初始只有头结点head,在循环单链表的构造方法中国需要通过head.next=head语句置为空表。
      2. 循环单链表中涉及查找操作时需要修改表尾的判断的条件。比如在遍历链表时，尾结点满足的条件是p.next==head。

  4. 循环双链表：

    1. 尾结点的指针指向头结点，头结点的prior指针指向尾结点。其特点就是表成为两个环。

    2. 特点：从表中任意一个结点出发均可以找到其他节点。通过头结点在O(1)时间内找到尾结点。

    3. 循环双链表的插入和删除与双链表的相同，所以两者的许多基本运算算法是相似的，只要区别就是：

      1. 初始只有头结点head,在循环双链表的构造方法中国需要通过dhead.prior=head和dhead.next = dhead语句置为空表。

    4. 循环双链表中涉及查找操作时需要修改表尾的判断的条件。比如在遍历链表时，尾结点满足的条件是p.next==head。

      class DLinkListClass { //双链表泛型类
    DLinkNode dhead; // 存放头结点

    public DLinkListClass() {
        dhead = new DLinkNode<>();
        dhead.prior = dhead;
        dhead.next = dhead;
    }
}

2.8、顺序表和链表的比较

  1. 逻辑结构
    1. 顺序表和链表都属于线性表，都是线性结构
  2. 存储结构
    1. 顺序表：顺序存储
      1. 优点：支持随机存取，存储密度高
      2. 缺点：大片连续空间分配方便，改变容量不方便
    2. 链表：链式存储
      1. 优点：离散的小空间分配方便，改变容量方便
      2. 缺点：不可随机存取，存储密度低
  3. 总结： 一般情况下，存储密度越高，空间利用率越高。仅从存储密度看，顺序表的存储空间利用率高。。
    1. 从空间方面看：顺序表需要预先分配连续的内存空间，容易出现上溢情况，需要扩展空间，会导致大量元素移动，效率低；链表的存储空间是动态分配的，只要内存有空间，就不会出现上溢。
    2. 从时间方面看：线性表的运算只要就是查找，很少做插入或删除操作，所以宜使用顺序表作为存储结构。

第三章：栈和队列(Stack and Queue)

3.1、栈(Stack)

  1. 栈的定义：
    • 栈是特殊的线性表：只允许在一端进行插入或删除操作，其逻辑结构与普通的线性表相同;
    • 栈顶(Top):允许进行插入和删除的一端(最上面的为栈顶元素)；
    • 栈底(Bottom): 固定的，不允许进行插入和删除的一端(最下面的为栈底元素)；
    • 空栈：不含任何元素的空表；
    • 特点：先进后出、后进先出；
    • 缺点：栈的大小不可变；

      3.2、栈的顺序存储结构及操作

  2. 顺序栈的四要素：
    1. 栈空的条件为top == -1;
    2. 栈满(栈上溢出)的条件为top == capacity-1,我们采用动态扩展容量方式，即栈满时，数组容量扩大两倍;
    3. 元素e进栈操作是先将栈顶指针top增1，然后再将元素e放在栈顶指针处;
    4. 出栈操作是先将栈顶指针top处的元素取出，然后将栈顶指针减1

这里我们区分一下正栈和倒栈的区别:
栈的存储空间S(1:m),可以理解Wie一个可以容纳m个数据的空间，本身看不出方法的，因此要根据数据进出栈时的指针变化来区分正栈和倒栈。
正栈：栈底为1，栈顶可以为1~m中任意一个数，但是top=m时，栈满，top不可能指向大于m的任何数字。此时，栈中的元素为top-bottom+1;，对于正栈，栈空的时候top可以为0、-1、···中任意一个数，可以理解每出一个栈，栈顶指针top-1,直到top=0，栈为空。

  • 特点：元素入栈：先将top值自增1，然后元素入栈，放在top指针处。data[++top]=e。自己的理解：想要在栈中添加一个数据，首先得知道栈中的容量够不够，如果直接塞进去，容量不够就会被挤出来，所以栈顶元素得先自增1，如果容量不过，则动态去扩展容量，这样就防止溢出。
  • 出栈操作：先将top指针的元素取出，然后栈顶指针再减1。e=data[top–]。这个就比较直接了当了，比如当我们在一摞书上，首先是先咋走一本书，总体书的数量才会再减1，不可能你都没有拿书，就开始减1吧。

倒栈： 初始状态下栈顶为m，栈底为m，当top=1时表示栈满，top不可能指向小于1的任何数。此时，栈中的元素数量等于m-top。可以理解为每出栈一个元素，top++，直到top=m+1,栈空。

  • 特点：元素入栈：先将元素入栈，然后top值自减1，放在top指针处。data[top–]=e。
  • 元素出栈：先将栈顶自增1，然后元素出栈。

总结：

  • 正栈 入栈：先增后入
  • 正栈 出栈：先出后减
  • 倒栈 入栈：先入后减
  • 倒栈 出栈：先增后出
class SqStackClass { // 顺序栈泛型类
    final int initcapacity = 10; // 初始栈的容量
    private int capacity; // 存放顺序栈的容量
    private E[] data; // 存放顺序栈的元素
    private int top; // 存放栈顶的指针

    public SqStackClass() {
        data = (E[]) new Object[initcapacity];
        capacity = initcapacity;
        top = -1;
    }

    public void UpdateCapacity(int newCapacity) { // 改变顺序栈的容量
        E[] newdata = (E[]) new Object[newCapacity];
        for (int i = 0; i <= top; i++) {
            newdata[i] = data[i];
        }
        capacity = newCapacity;
        data = newdata;
    }

    public boolean IsEmpty() { // 判断栈是否为空
        return top == -1;
    }

    public void push(E e) { // 元素进栈
        if (top == capacity) {
            UpdateCapacity( 2 * (top + 1) );
        }
        top++;
        data[top] = e;
    }

    public E pop(){ // 出栈操作
        if(IsEmpty()){
            throw new IllegalArgumentException("栈空");
        }
        E e=data[top];
        top--;
        return e;
    }

    public E peek(){
        if(IsEmpty()){
            throw new IllegalArgumentException("栈空");
        }
        return data[top];
    }
    // 从上面的算法来看，所有的操作时间复杂度都是O(1)
}

3.3、栈的链式存储结构及操作

  1. 定义：采用链式存储的栈称为链栈

  2. 优点：便于多个栈共享存储空间和提高效率，且不存在栈满上溢的情况。

  3. 特点：

    1. 进栈和出栈都只能在栈顶一端进行(链头作为栈顶)

  4. 所以，链栈实际就是一个只能采用头插法插入或删除数据的链表。

     // 在此说明一下，结点类在前面的单链表中定义过了。
 class LinkStackClass {
     LinkNode head;

     public LinkStackClass() {
         head = new LinkNode<>();
         head.next = null;
     }

     public boolean empty() { // 判断链栈是否为空
         return head.next == null;
     }

     public void push(E e) { // 入栈
         LinkNode s = new LinkNode();
         s.next = head.next;
         head.next = s;
     }

     public E pop() { // 出栈
         if (empty()) {
             throw new IllegalArgumentException( "栈空" );
         }
         E e = (E) head.next.data;
         head.next = head.next.next;
         return e;
     }

     public E peek() { // 取栈顶元素
         if (empty()) {
             throw new IllegalArgumentException( "栈空" );
         }
         return (E) head.next.data;
     }

 }

3.4、队列(Queue)

  1. 定义：队列简称对，是一种操作受限的线性表，只允许在表的一端进行插入，而在表的另一端进行删除。
  2. 特点：
    1. 操作特性：先进先出
    2. 队头：允许删除的一端
    3. 队尾：允许插入的一端
    4. 空队列：不含任何元素的空表

3.5、队列的顺序存储结构及操作

  1. 顺序队分为非循环队和循环队两种方式。首先我们先说非循环队及特征
  2. 默认初始值front和rear均为 -1
  3. 队头指针为front(实际上是对头元素的前一个位置)，队尾指针为rear(正好是队尾元素的位置).为什么让front指向队列中前一个元素位置？因为在front增1后，该位置的元素已经出队了
  4. 队满：rear == MaxSize-1
  5. 元素每次进队：rear = rear+1(队尾先+1，再入队)
  6. 元素每次出队：front = front+1(队头先+1，再出队)
  7. 队空的条件：front == rear
  8. 特点：非循环队列会出现假溢出。相关操作的时间复杂度都是O(1)
    class SqQueueClass { // 非循环队列的泛型类
        final int MaxSize = 10; // 假设容量为10
        private E[] data; // 存放队列中的元素
        private int front, rear; // 队头、队尾的指针

        public SqQueueClass() {
            data = (E[]) new Object[MaxSize];
            front = rear = -1; // 队头、队尾的指针默认为-1，队为空
        }


        public boolean empty() { // 判断是否为空
            return front == rear;
        }

        public void push(E e) { // 入队操作
            if (rear == MaxSize - 1) {
                throw new IllegalArgumentException( "队满" );
            }
            rear++;
            data[rear] = e;
        }

        public E pop() { // 出队
            if (front == rear) {
                throw new IllegalArgumentException( "队空" );
            }
            front++;
            E e = data[front];
            return e;
        }

        public E peek() { // 取头元素
            if (front == rear) {
                throw new IllegalArgumentException( "队空" );
            }
            return data[front + 1];
        }
    }

3.5.1、队列的顺序存储结构及操作

  1. 这里来说循环队列的特点和操作
  2. 定义：将循环队列想象为一个环状的空间，即把存储队列元素的表从逻辑上视为一个环，称为循环队列。
  3. 主要作用：为了克服非循环队列的假溢出
  4. 这里以下，代码及以上，重点！！！！:
    1. 因为循环队列首尾相连，但队尾指针rear=MaxSize-1时，再前进一个位置就是0位置。所以我们可以利用求余(%)实现进队和出队的所在位置：
      1. 队首指针循环进1：front=(front+1)%MaxSize
      2. 队尾指针循环进1：rear=(rear+1)%MaxSize
    2. 队空的条件：rear==front
    3. 队满条件为(rear+1)%MaxSize == front
    4. 进队：rear=(rear+1)%MaxSize，再将元素e放在该位置上
    5. 出队：front=(front+1)%MaxSize，取出该元素的位置
    6. 队中的元素个数：(rear-front+MaxSize)%MaxSize
    7. 循环队列和非循环队列的头尾指针值不一样，循环队列为front=rear=0
    ```Java
class CSqQueueClass<E> {
    final int MaxSize = 10; // 假设容量为10
    private E[] data; // 存放队列中的元素
    private int front, rear; // 队头、队尾的指针

    public CSqQueueClass() {
        data = (E[]) new Object[MaxSize];
        front = rear = 0;
    }

    public boolean empty() { // 判断是否为空
        return rear == front;
    }

    public void push(E e) { // 入队
        if ((rear + 1) % MaxSize == front) {
            throw new IllegalArgumentException( "队满" );
        }
        rear = (rear + 1) % MaxSize;
        data[rear] = e;
    }

    public E pop() { // 出队
        if (empty()) {
            throw new IllegalArgumentException( "队空" );
        }
        front = (front + 1) % MaxSize;
        return data[front];
    }

    public E push() { // 返回队头元素
        if (empty()) {
            throw new IllegalArgumentException( "队空" );
        }
        return data[(front + 1) % MaxSize];
    }
}
    ```
